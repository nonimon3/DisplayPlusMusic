# DisplayPlusMusic Spotify認証 不具合 調査メモ

## 症状
- EVEN Hub上でDisplayPlusMusic起動 → Client ID/Secret入力 → "Save & Authorize" 押下
- アラート「Save pressed」表示後、それ以上進まない（Spotify認証ページに遷移しない）
- Spotifyはプレミアム

## 関連ソース
- `src/presenter/viewPresenter.ts:89-103` `saveAndAuthorize()`
  - L90: `alert("Save pressed")` ← デバッグ用アラート（必ず最初に出る）
  - L99-100: `await storage.setItem(...)` でクレデンシャル保存
  - L102: `spotifyAuthModel.generateAuthUrl(clientId)` 呼び出し
- `src/model/spotifyAuthModel.ts:27-44` `generateAuthUrl()`
  - L43: `window.location.href = authUrl.toString();` ← ここで Spotify 認可ページへ遷移しようとする
- `src/utils/storage.ts` … EvenAppBridge.setLocalStorage を経由（await されていないので即時 return）
- `app.json` の network 許可: `https://accounts.spotify.com/*` は fetch 用（top-level navigation には別ルール）
- リダイレクトURI 固定: `https://oliemanq.github.io/DisplayPlusMusic/`

## 根本原因（推定）
EVEN Hub の WebView で `window.location.href = "https://accounts.spotify.com/authorize?..."` の **トップレベル遷移が機能していない** 可能性が高い。

理由:
1. EVEN Hub アプリは WebView 内で動作するスタンドアロン Web アプリで、外部 URL への top-level 遷移はホスト側ポリシーで弾かれることが多い
2. たとえ遷移できても、認可後 Spotify が `https://oliemanq.github.io/DisplayPlusMusic/` にコールバックする → packaged 起動の場合 origin が一致せず元のアプリ状態に戻れない
3. PKCE ではなく Implicit/Authorization Code (Client Secret 同梱) フローを使っており、SPA 内でのリダイレクト型認可を前提にしている

## 切り分け手順（ユーザに依頼するもの）
1. **モバイルブラウザで直接試す**:
   - https://oliemanq.github.io/DisplayPlusMusic/ をスマホ Chrome/Safari で開く
   - 同じく Client ID/Secret 入れて Save & Authorize → ここでも止まるか？
   - もしブラウザでは Spotify ログインに飛べる → EVEN Hub の WebView が原因
   - もしブラウザでも止まる → コード側の問題（`window.location.href` 自体が呼ばれていない可能性、storage await ハング等）

2. **Spotify Developer Dashboard 設定確認**:
   - 自分のアプリの Redirect URI 欄に **完全一致** で `https://oliemanq.github.io/DisplayPlusMusic/`（末尾スラッシュ含む）が登録されているか
   - "Web API" にチェックが入っているか

3. **DevTools / リモートデバッグ**:
   - Console に `Using Redirect URI: ...` ログが出ているか確認
   - 出ていれば storage 通過済 → 遷移段階で失敗
   - 出ていなければ storage.setItem で詰まっている

## 想定される改修案
A. 外部ブラウザを開く方式に切替
   - `window.open(authUrl, '_blank')` あるいは EVEN Hub SDK が外部 URL オープン API を提供していればそれを使う
B. PKCE フロー化
   - Client Secret を排し PKCE で SPA 完結型に
C. デバッグアラート除去 + try/catch ログ追加
   - `alert("Save pressed")` を消し、`try { await ... } catch(e) { alert(e) }` に変更してエラー把握
